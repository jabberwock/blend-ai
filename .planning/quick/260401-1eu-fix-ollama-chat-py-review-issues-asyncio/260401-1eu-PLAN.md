    Fix 1 -- Cached event loop in execute_tool (lines 151-174):
    - Add `self._loop: asyncio.AbstractEventLoop | None = None` in __init__
    - In execute_tool, lazily create the loop on first call:
      ```python
      if self._loop is None:
          self._loop = asyncio.new_event_loop()
      ```
    - Replace `asyncio.run(mcp.call_tool(name, arguments))` with
      `self._loop.run_until_complete(mcp.call_tool(name, arguments))`
    - Add cleanup in close(): if self._loop, close it and set to None
    - Add `import asyncio` at module top level (move from inside execute_tool)

    Fix 2 -- Tighten Pattern 2 JSON scanner (line 386):
    - Change the condition from:
      `if isinstance(obj, dict) and "name" in obj:`
    - To:
      `if isinstance(obj, dict) and "name" in obj and "arguments" in obj:`
    - This prevents false positives from Blender scene data like {"name": "Cube", "vertices": 8}

    Fix 3 -- Strip opening <tool_call> tags in _strip_tool_markup (lines 439-445):
    - Add a regex BEFORE the </tool_call> removal:
      `cleaned = re.sub(r'<tool_call>', '', cleaned)`
    - This ensures both opening and closing tags are removed

    Fix 4 -- Structured vision note in chat() (lines 294-297):
    - Replace:
      ```python
      self.messages.append({
          "role": "tool",
          "content": result + vision_note,
      })
      ```
    - With:
      ```python
      if vision_note:
          try:
              result_obj = json.loads(result)
              result_obj["vision_analysis"] = analysis
              tool_content = json.dumps(result_obj)
          except (json.JSONDecodeError, TypeError):
              tool_content = result + vision_note
      else:
          tool_content = result
      self.messages.append({
          "role": "tool",
          "content": tool_content,
      })
      ```
    - Note: use `analysis` (the raw analysis string) not `vision_note` (which has the prefix) for the JSON key value. The `analysis` variable is already in scope from the `if` block above.
  </action>
